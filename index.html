Welcome to Mosaic Website’s documentation!¶

Introduction¶

This project is a mosaic website built with Firebase.

Project Structure¶

  • functions/: Firebase Cloud Functions (Node.js).

  • public/: Static frontend assets (HTML, CSS, JS).

  • scripts/: Maintenance and migration scripts.

Firebase Functions¶

getHymnIndex(request)¶

A Callable Cloud Function that fetches the entire hymn index from Firestore.

This function returns a simplified list of hymns with basic metadata, optimized for search and display in the frontend lookup tool. Results are cached in memory for 5 minutes.

Arguments:

  • request (Object) – The request object.

  • id (string) – The Firestore document ID.

  • hymn_name (string) – The title of the hymn.

  • variations (number) – The number of available versions/arrangements.

  • music_writer (string) – The composer of the music.

  • lyrics_writer (string) – The author of the lyrics.

  • last_played_date (string|null) – The last date the hymn was played.

  • tags (Array.<string>) – Descriptive tags for the hymn.

  • database_url (string) – Relative URL to the hymn details page.

Returns:

Promise.<Array.<Object>> – A promise that resolves to an array of hymn objects.

Frontend Components¶

Main Lookup (main.js)¶

toggleTag(tag)¶

Toggles a tag in the selectedTags filter list.

Arguments:

  • tag (string) – The tag to toggle.

performSearch()¶

Filters the hymns based on the current searchQuery and selectedTags.

selectHymn(id)¶

Redirects the user to the details page for a specific hymn.

Arguments:

  • id (string) – The Firestore ID of the hymn.

Hymn Details (hymn-details.js)¶

renderHymnDetails(hymn)¶

Renders the hymn’s details into the DOM.

Arguments:

  • hymn (Object) – The hymn data object from Firestore.

updateLastTimePlayed(button)¶

Updates the ‘last_played_date’ field for the current hymn in Firestore. Sets the date to the upcoming Sunday.

Arguments:

  • button (HTMLElement) – The button element that triggered the update.

copyToClipboard(text, button)¶

Copies the provided text to the system clipboard.

Arguments:

  • text (string) – The text to copy.

  • button (HTMLElement) – The button element to provide visual feedback.

copyImageToClipboard(imageUrl, button)¶

Fetches an image from a URL and copies it to the clipboard as a PNG.

Arguments:

  • imageUrl (string) – The URL of the image to copy.

  • button (HTMLElement) – The button element to provide visual feedback.

Hymn Manager (manager.js)¶

showToast(message, type="'success'")¶

Displays a temporary toast message to the user.

Arguments:

  • message (string) – The message to display.

  • type (string) – The type of toast (‘success’ or ‘error’).

loadHymns()¶

Loads the list of all hymns from Firestore, ordered by name.

loadTags()¶

Loads all available tags from the ‘tags’ collection.

startEditHymn(hymn)¶

Populates the form with data from an existing hymn to begin editing.

Arguments:

  • hymn (Object) – The hymn object to edit.

deleteHymn(hymn)¶

Deletes a hymn from Firestore and its associated images from Firebase Storage.

Arguments:

  • hymn (Object) – The hymn object to delete.

handleSubmit()¶

Validates and submits the hymn form to Firestore and Firebase Storage. Handles both creation of new hymns and updates to existing ones.

Maintenance Scripts¶

checkDuplicates()¶

Fetches all hymns from Firestore and logs any duplicates found by name.

cleanDuplicates()¶

Identifies and deletes duplicate hymns. Can be run with –force to skip confirmation.

migrateHymns()¶

Reads the ‘hymn_index.json’ file and performs a batch write to Firestore. Each hymn is assigned a default “Default” version.
